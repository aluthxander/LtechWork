Diagram yang Perlu Dibuat
* Use Case Diagram
    Untuk menggambarkan fungsionalitas sistem dari sudut pandang pengguna.
    Contoh use case:
    - Admin mengelola data karyawan
    - Admin memproses gaji
    - Karyawan melihat slip gaji
    - Admin mengatur potongan dan tunjangan
* Entity Relationship Diagram (ERD)
    Untuk merancang struktur database payroll seperti:
    - Tabel karyawan, gaji, tunjangan, potongan, absensi, slip gaji
    - Relasi antara tabel

* Class Diagram
    Untuk menggambarkan struktur model dan relasi antar class Laravel (Eloquent Model).

* Activity Diagram
    Untuk menunjukkan alur proses tertentu, misalnya:
    - Proses perhitungan gaji
    - Proses generate slip gaji

* Sequence Diagram
    Menjelaskan alur komunikasi antar komponen, cocok untuk menjelaskan interaksi antara frontend dan backend saat aksi tertentu, seperti login atau generate slip.

* Component Diagram (opsional)
    Menjelaskan bagaimana komponen besar seperti Laravel API, React App, dan Database saling berinteraksi.

* Data Flow Diagram
    perlu dibuat, terutama jika kamu ingin menggambarkan alur data yang mengalir antara entitas luar, proses, dan penyimpanan data dalam sistem payroll secara lebih detail dan sistematis.

🔍 Website Riset & Inspirasi UI/UX Sistem Payroll
1. 📌 Dribbble
🔎 Cari dengan kata kunci seperti: payroll system, HR dashboard, salary management UI.

🖼 Banyak contoh desain modern dan profesional.

Cocok untuk referensi visual dan layout.

2. 🎨 Behance
Platform portofolio dari desainer profesional.

Sering menampilkan desain lengkap: mulai dari wireframe, UI, UX flow, hingga studi kasus.

Contoh keyword pencarian: Payroll Dashboard, HRIS System, Finance Admin UI.

3. 📚 UI Garage
Kumpulan elemen UI dari berbagai aplikasi nyata.

Bisa filter berdasarkan kategori seperti "Admin Dashboard", "Finance", dll.

Bagus untuk melihat desain spesifik komponen seperti form penggajian, tabel gaji, dsb.

4. 🧠 UX Archive
Fokus pada alur UX aplikasi nyata, seperti onboarding, login, pembayaran, dll.

Berguna untuk mempelajari alur pengguna (UX flow) secara logis.

Bisa kamu adaptasi untuk flow sistem payroll: login → input gaji → generate slip → laporan.

5. 💡 Mobbin
Koleksi desain mobile app real-world dari aplikasi populer.

Cocok jika sistem payroll-mu juga akan diakses lewat mobile.

Bisa lihat cara aplikasi seperti Gusto, Zenefits, atau Paychex menangani UI/UX-nya.

6. 🧩 Collect UI
Update harian inspirasi desain UI berdasarkan kategori.

Cari bagian “Dashboard”, “Table”, atau “Admin Panel”.

7. 📁 SaaS Landing Page
Meski lebih fokus pada landing page, tapi banyak sistem payroll adalah SaaS.

Bisa jadi inspirasi untuk halaman utama sistem payroll (jika berbasis web app).
